Get it out of your head.

A local-first scratchpad for tasks, notes, lists and timers — running as a Home Assistant App. A local model you already run does the parsing. Your data stays on your box.

Install on Home Assistant ↓
The LLM only parses and ranks what you typed — it never invents data, and it never acts on your house. House actions (a timer ringing a speaker) are deterministic code you opt into, not model output. No cloud by default, no telemetry.

Install on Home Assistant

Fanad ships as a Home Assistant App (add-on) — a prebuilt image the Supervisor runs for you. You need Home Assistant OS or Supervised (the Container/Core types don't have the add-on store — see plain Docker below).

1

Add the Fanad repository

In Home Assistant: Settings → Add-ons → Add-on store (newer installs: Settings → Apps) → the ⋮ menu → Repositories, and add this URL:

https://github.com/NTBooks/fanad-ha

Add via My Home Assistant →

2

Install & start

Find Fanad in the store, click Install, then Start. The first start pulls a small prebuilt image (works on both 64-bit Pi / aarch64 and amd64).

3

Open the Web UI

Click Open Web UI (or the Fanad sidebar panel). Text a snippet in the chat, watch it get filed, and ask "what should I do?". Then connect a model.

Connect a model

Fanad uses a local model you already run — LM Studio or Ollama. Open ⚙ Settings, pick the provider, and enter its address.

Use a LAN IP, not localhost. Fanad runs in a container, so localhost points at the container — not your desktop. Point it at the machine running the model, e.g. http://192.168.1.50:1234/v1 (LM Studio) or http://192.168.1.50:11434/v1 (Ollama).

No model yet? Set llm_provider to mock in the add-on's Configuration tab to try the whole flow with a built-in stub. A Pi runs Fanad happily; the model should live on a real box (the desktop already running LM Studio/Ollama).

Let it ring the house

Fanad's timers and reminders can ring your Home Assistant speakers, and ha <command> passes straight to your Assist agent. As an add-on it's already paired with your Home Assistant — no URL or token to paste.

  • Enable the Home Assistant module — Settings, or type optin ha in chat.
  • Pick which speakers, scripts, or notifications to ring in Settings → Home Assistant.

Backups & uninstalling

Uninstalling the add-on deletes its data. Fanad stores everything in /data, which the Supervisor wipes on uninstall. Back up first.

Home Assistant's native backups include the add-on's data, so a normal HA backup captures your Fanad database and its encryption key together — restore brings both back.

Prefer plain Docker?

The exact same image runs anywhere — for the Container/Core install types or any server:

docker run -p 8787:8787 -v fanad:/data ghcr.io/ntbooks/fanad-app:latest

Then open http://<host>:8787. The add-on also exposes an optional direct 8787/tcp port for the same UI/API or the fanad CLI client.
